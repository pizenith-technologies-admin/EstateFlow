# Development Guide

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Node.js | 20.x | [nodejs.org](https://nodejs.org) |
| npm | 10.x | Bundled with Node |
| Git | any | [git-scm.com](https://git-scm.com) |

---

## Local Setup

**1. Clone and install**
```bash
git clone https://github.com/gunnishmehta/EstateFlow_Deployment.git
cd EstateFlow
npm install
```

**2. Set up environment variables**
```bash
cp .env.example .env
```

Open `.env` and fill in at minimum:
```
DATABASE_URL=postgresql://...   # From neon.tech
JWT_SECRET=<random 48-char string>
SESSION_SECRET=<different random string>
```

The app works without Cloudinary/SendGrid/GCS in development — those features will log warnings but won't crash.

**3. Push schema to database**
```bash
npm run db:push
```

**4. Start the dev server**
```bash
npm run dev
```

Visit `http://localhost:5000` — the React frontend and API are served together.

## Project Layout in Detail

```
server/
├── app.ts           # Express app creation + middleware (used by Vercel)
├── index.ts         # Local dev entry: imports app, adds Vite, calls listen()
├── routes.ts        # All ~100 API route handlers
├── simpleAuth.ts    # Login/register/logout + isAuthenticated middleware
├── storage.ts       # All database queries (3,000+ lines, one function per operation)
├── db.ts            # Neon pool + Drizzle instance
├── cloudinary.ts    # Image upload service
├── emailService.ts  # SendGrid email templates
├── objectStorage.ts # Google Cloud Storage
├── objectAcl.ts     # GCS object access control
├── seedData.ts      # Sample data for development
└── vite.ts          # Vite dev server integration (dev only)

shared/
└── schema.ts        # Drizzle table definitions + Zod validation schemas
                     # This is the single source of truth for all types

api/
├── _handler.ts      # Vercel handler source — EDIT THIS for server changes
└── index.js         # Pre-built bundle (generated by esbuild, committed to git)
```

> **Important**: When making server changes, edit files in `server/`. The `api/index.js` bundle is regenerated automatically on Vercel deploy via `build:vercel`. For local dev, `server/index.ts` is the entry point — it never touches `api/`.

## Making Server Changes

### For local development
Edit any file in `server/` and the dev server auto-reloads (tsx watch mode).

### For Vercel deployment
Server changes go live automatically when you push to `main` — Vercel runs `npm run build:vercel` which rebuilds `api/index.js`.

If you want to test the production bundle locally first:
```bash
npx esbuild api/_handler.ts --bundle --platform=node --format=esm --packages=external --outfile=api/index.js
```

## Adding a New API Route

1. Open `server/routes.ts`
2. Add your route using the Express app instance:
```ts
app.get("/api/my-endpoint", isAuthenticated, async (req, res) => {
  try {
    const result = await storage.myQuery();
    res.json(result);
  } catch (error) {
    res.status(500).json({ message: "Error" });
  }
});
```
3. If you need a new database query, add it to `server/storage.ts`
4. If you need a new table, add it to `shared/schema.ts` then run `npm run db:push`

---

## Adding a New Database Table

1. Open `shared/schema.ts`
2. Define the table using Drizzle:
```ts
export const myTable = pgTable("my_table", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  userId: text("user_id").references(() => users.id),
  createdAt: timestamp("created_at").defaultNow(),
});
```
3. Export the types:
```ts
export type MyTable = typeof myTable.$inferSelect;
export type InsertMyTable = typeof myTable.$inferInsert;
```
4. Apply to database:
```bash
npm run db:push
```

## Database Access Pattern

All DB queries go through `server/storage.ts`. Don't query the database directly from routes — use or add a storage method.

```ts
// In storage.ts
async getUser(id: string): Promise<User | undefined> {
  const result = await db.select().from(users).where(eq(users.id, id));
  return result[0];
}

// In routes.ts
const user = await storage.getUser(userId);
```

## Mobile Development

```bash
cd mobile

npx expo start
```

Scan the QR code with **Expo Go** on your phone. The app connects to `localhost:5000` by default in dev mode.

1. Create file in `mobile/src/screens/<role>/MyScreen.tsx`
2. Add to the navigation type in `mobile/src/navigation/types.ts`
3. Register in the appropriate tabs file (`AgentTabs.tsx`, etc.)

## Common Issues

**`DATABASE_URL must be set`**
→ Copy `.env.example` to `.env` and fill in `DATABASE_URL` from neon.tech

**`Cannot find module './routes'` on Vercel**
→ This is fixed by the esbuild bundling approach. If it recurs, rebuild: `npx esbuild api/_handler.ts --bundle ... --outfile=api/index.js` and commit

**Cloudinary uploads failing**
→ Set `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET` in `.env`

**Mobile app shows "Network error"**
→ Check `EXPO_PUBLIC_API_URL` in `mobile/.env`. In dev, should be `http://localhost:5000`. Use your machine's local IP (not `localhost`) when testing on a physical device: `http://192.168.x.x:5000`

**Expo build fails on "Install dependencies"**
→ Check `mobile/package.json` for version conflicts. Run `cd mobile && npm install` locally and fix any peer dependency warnings first.
