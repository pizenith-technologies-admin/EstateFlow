*(Copy/paste this into Replit AI. Extend only; do not regress Agent/Client features.)*

You are adding a third, read-mostly portal for **Brokerages** to **monitor** their agents and agent-client activity. Brokerage users **do not create tours**; they audit performance, coach agents, and export reports. Agents can be **linked to a brokerage now or later**; agents may also operate standalone.

---

## 0) Guardrails

* Preserve stack, routes, styles; add only.
* **RBAC** must support 3 roles: `brokerage`, `agent`, `client`. Enforce at API + UI.
* CAD, km, ISO/24h.
* PII safe: never expose sensitive IDs (SIN, bank numbers, DL), including nested objects.
* Broker **cannot** create/edit tours, itineraries, or reminders.

## 3) Dashboard (Brokerage)

## 11) Data Model (additions)

```sql
Brokerage (
  id PK,
  name TEXT NOT NULL,
  logoUrl TEXT,
  contactEmail TEXT,
  contactPhone TEXT,
  settings JSONB,         -- permissions, comms, branding for broker portal
  createdAt TIMESTAMP
);

BrokerageAgent (
  id PK,
  brokerageId FK REFERENCES Brokerage(id),
  agentId FK REFERENCES Agent(id),
  role TEXT DEFAULT 'member', -- member/manager
  createdAt TIMESTAMP,
  UNIQUE(brokerageId, agentId)
);

CoachingNote (
  id PK,
  brokerageId FK,
  agentId FK,
  authorId FK,            -- brokerage user
  note TEXT NOT NULL,
  createdAt TIMESTAMP
);

-- Optional: Teaming
BrokerageTeam (
  id PK,
  brokerageId FK,
  name TEXT NOT NULL,
  createdAt TIMESTAMP
);
BrokerageTeamAgent (
  id PK,
  teamId FK REFERENCES BrokerageTeam(id),
  agentId FK REFERENCES Agent(id),
  createdAt TIMESTAMP,
  UNIQUE(teamId, agentId)
);
```

> Reuse existing `Client`, `Tour`, `Offer`, `Exception`, `RequirementVersion`, `Rating` entities; queries should aggregate by `agentId` constrained to the broker’s linked agents.

## 12) API (RBAC enforced)

* `GET /api/broker/kpis?from=&to=&agentId=&teamId=`
* `GET /api/broker/agents?search=&teamId=`
* `GET /api/broker/agents/:agentId` (overview + tabs data)
* `GET /api/broker/clients?search=&agentId=&type=&status=`
* `GET /api/broker/clients/:clientId`
* `GET /api/broker/reports?scope=all|agent|client&params...` → returns all metrics shown
* `GET /api/broker/documents?agentId=&clientId=&q=`
* `GET /api/broker/media?agentId=&clientId=&from=&to=`
* `POST /api/broker/agents/link` `{ email }` (invite/link by agent email)
* `DELETE /api/broker/agents/:agentId/link`
* `POST /api/broker/coaching-notes` `{ agentId, note }`
* `GET /api/broker/audit?from=&to=&user=`
* `PUT /api/broker/settings` (profile, comms, permissions, scheduled reports)

> All responses must keep existing sanitization rules (no sensitive PII).
