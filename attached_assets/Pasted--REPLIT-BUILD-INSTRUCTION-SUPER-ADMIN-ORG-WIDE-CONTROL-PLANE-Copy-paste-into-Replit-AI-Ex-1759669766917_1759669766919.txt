**REPLIT BUILD INSTRUCTION — SUPER ADMIN (ORG-WIDE CONTROL PLANE)**
*(Copy/paste into Replit AI. Extend only; no regressions to existing Brokerage / Agent / Client portals.)*

You are adding a **fourth role: `superadmin`** with a **global control plane** at `/admin`. Super Admin can **see everything**, **create accounts**, **link accounts**, and **jump into** any Brokerage, Agent, or Client context in **one click**. They do **not** run tours; they manage the organization.

---

## 0) RBAC (authoritative)

Support **four roles** and enforce on API + UI:

* `superadmin` – full read of all data + account lifecycle actions (create/suspend/reactivate/link/impersonate).
* `brokerage` – manage their brokerage profile and **link/unlink agents**; **cannot** create tours; **read** analytics across their linked agents.
* `agent` – manage their book of business; can create **clients only under themselves**.
* `client` – their own experience only.

**Creation rules**

* **Super Admin** can create: **Brokerage accounts** and **Agent accounts** (and assign an Agent either as **Brokerage Agent** under a brokerage or as an **Independent Agent**).
* **Brokerage** can **create/link Agents** (brokerage agents) and **cannot** create clients directly (clients belong to agents).
* **Agent** can **create Clients** under themselves only.

## 8) Data Model (adds/adjustments)

```sql
-- Users table already exists with role field; ensure enum includes 'superadmin'.

Brokerage (
  id PK,
  name TEXT NOT NULL,
  logoUrl TEXT NULL,
  contactEmail TEXT NULL,
  contactPhone TEXT NULL,
  createdAt TIMESTAMP,
  updatedAt TIMESTAMP,
  status TEXT DEFAULT 'active' -- active/suspended
);

BrokerageAgent (
  id PK,
  brokerageId FK REFERENCES Brokerage(id),
  agentId FK REFERENCES User(id), -- role=agent
  createdAt TIMESTAMP,
  UNIQUE (brokerageId, agentId)
);

-- Optional mapping for Independent: agent without a BrokerageAgent row.

ImpersonationLog (
  id PK,
  actorId FK REFERENCES User(id), -- superadmin
  targetType TEXT CHECK (targetType IN ('brokerage','agent','client')),
  targetId TEXT NOT NULL,
  mode TEXT DEFAULT 'read', -- read/readwrite (start with read)
  startedAt TIMESTAMP,
  endedAt TIMESTAMP
);
```

Re-use existing entities (Client, Tour, Offer, RequirementVersion, Rating, Exception). Aggregate queries must support filters by brokerage → agents → clients.

## 9) API (RBAC enforced)

Admin-only endpoints:

* `GET /api/admin/kpis?from=&to=`
* **Brokerages**

  * `GET /api/admin/brokerages?search=&status=`
  * `POST /api/admin/brokerages`
  * `GET /api/admin/brokerages/:id`
  * `PUT /api/admin/brokerages/:id`
  * `POST /api/admin/brokerages/:id/suspend`
  * `POST /api/admin/brokerages/:id/reactivate`
  * `POST /api/admin/brokerages/:id/agents/link` `{ agentId }`
  * `DELETE /api/admin/brokerages/:id/agents/:agentId`
* **Agents**

  * `GET /api/admin/agents?search=&brokerageId=&independent=`
  * `POST /api/admin/agents` (create + assign)
  * `PUT /api/admin/agents/:id` (reassign brokerage/independent)
  * `POST /api/admin/agents/:id/suspend|reactivate`
  * `POST /api/admin/agents/:id/reset-invite`
* **Clients**

  * `GET /api/admin/clients?search=&brokerageId=&agentId=&type=&status=`
  * `GET /api/admin/clients/:id`
  * `POST /api/admin/clients/:id/suspend|reactivate`
  * `POST /api/admin/clients/:id/reset-invite`
* **Reports**

  * `GET /api/admin/reports?...` (returns all sections)
  * `POST /api/admin/reports/schedule` (weekly/monthly)
* **Impersonation**

  * `POST /api/admin/jump` `{ targetType, targetId }` → returns signed impersonation token + redirect URL
  * `POST /api/admin/return` (end session; log)
* **Audit**

  * `GET /api/admin/audit?from=&to=&actor=&targetType=&targetId=`

All responses must honor existing **sanitization rules** (no SIN, bank numbers, DL; nested cleansing).

## 11) Acceptance Criteria

* Super Admin can:

  * Create **Brokerage** and **Agent** accounts; assign agents as **Brokerage Agents** or **Independent**.

  * See org-wide KPIs and drilldowns by brokerage/agent/client.

  * Export Reports CSV/PDF; schedule email reports.
* Brokerage can link/create agents; Agents can create Clients; Agents/Clients cannot escalate roles.
* RBAC blocks non-admin access to `/admin` and Admin actions.
* No regressions to existing portals or security posture.
