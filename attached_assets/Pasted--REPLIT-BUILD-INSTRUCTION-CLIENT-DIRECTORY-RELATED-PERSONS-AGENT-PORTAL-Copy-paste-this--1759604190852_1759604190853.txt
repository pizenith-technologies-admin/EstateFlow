## 7) Data Model (add tables)

```sql
-- Contacts
Contact (
  id PK,
  fullName TEXT NOT NULL,
  dateOfBirth DATE NULL,
  phones JSONB DEFAULT '[]',   -- [{label, number}]
  emails JSONB DEFAULT '[]',   -- [{label, address}]
  notes TEXT,
  hasApp BOOLEAN DEFAULT FALSE,
  lastActiveAt TIMESTAMP NULL,
  createdAt TIMESTAMP,
  updatedAt TIMESTAMP
);

-- Link a contact to a primary client with a role
ClientContactLink (
  id PK,
  clientId FK REFERENCES Client(id),
  contactId FK REFERENCES Contact(id),
  relationshipType TEXT NOT NULL,  -- enum values listed above
  isPrimary BOOLEAN DEFAULT FALSE, -- mark the main client’s own record if needed
  createdAt TIMESTAMP,
  updatedAt TIMESTAMP,
  UNIQUE (clientId, contactId)
);
```

> If a “Primary Client” should also live in `Contact`, create a shadow contact row flagged `isPrimary=true` and link it to itself for consistent querying.

## 8) API Endpoints

* `POST  /api/directory/contacts/:contactId/invite` *(create login/invite if allowed)*

* `POST  /api/directory/contacts/:contactId/birthday-reminder` *(toggle on/off)*

All endpoints RBAC-protected (Agent). Sanitize responses (no sensitive PII).
